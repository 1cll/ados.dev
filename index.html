よくある質問

導入前に気になるポイントをまとめました

GitHub Copilot Coding Agent とは何が違いますか？

Copilot Coding Agent は「Issue を受けてコードを書く 1 人のジュニア開発者」です。ADOS は Copilot を含む複数の AI を束ねて「チーム全体の開発運用を自律的に回す」プラットフォームです。SRE 監視、CI/CD 自動修復、バックログ自動生成、AutoPilot による自律改善など、Copilot にはない機能を多数搭載しています。

GitHub 以外の VCS でも使えますか？

はい。GitHub、GitLab、Bitbucket に対応しています。既存のソース管理を変えることなく導入可能です。

自社のインフラで動かせますか？

はい。セルフホスト Docker Runner を使えば、自社のオンプレミス環境やプライベートクラウドで AI エージェントを実行できます。コードが外部に出ることはありません。

AutoPilot とは何ですか？

AutoPilot は ADOS の自律改善モードです。有効にすると、AI がリポジトリのコードを定期的に分析し、改善項目を自動で Issue 化・実装します。人間は承認するだけで、リポジトリが継続的に改善されていきます。

どの AI エージェントが使われるか選べますか？

はい。Issue の複雑さに応じた自動選択のほか、手動で指定することも可能です。Copilot CLI、Claude Code、Codex に対応しています。

セキュリティは大丈夫ですか？

セルフホスト Runner によりコードが外部に出ることはありません。また、全ワークフローに Plan → Approval → Implement の承認ゲートを設定でき、AI が勝手にデプロイすることはありません。
